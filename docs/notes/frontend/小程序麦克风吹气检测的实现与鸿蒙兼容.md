# 小程序麦克风吹气检测的实现与鸿蒙兼容

## 背景

需求是做一个"对着手机吹气触发动画"的交互。听起来很酷，但真正做起来才发现：麦克风接收到的不只是吹气——说话、拍手、环境噪音都会让音量飙升。如果只判断音量，用户随便咳嗽一下就触发了。

更头疼的是，开发完在 iPhone 和 Pixel 上测得好好的，拿到华为手机上一测——吹到缺氧都不触发。原来鸿蒙系统的降噪太激进，把吹气声直接过滤掉了。

---

## 核心方案：双重阈值判断

**音量阈值 + 低频能量占比** = 可靠的吹气识别。

吹气的频率特征是：**低频能量占比高**（0-500Hz 占总能量的 30%~60%）。而说话、拍手的低频占比通常只有 10%~25%。

```javascript
const isBlowing = (rmsVolume >= volumeThreshold) &&   // 够响
                  (lowFreqRatio >= lowFreqThreshold)   // 且是低频（吹气特征）
```

典型数值：

| 声音类型 | RMS 音量 | 低频占比 | 是否触发 |
|---------|---------|---------|:---:|
| 环境噪音 | 0.01 | 5~20% | ❌ |
| 说话 | 0.12 | 15~35% | ❌ |
| 拍手 | 0.25 | 10~25% | ❌ |
| 吹气 | 0.15 | 30~60% | ✅ |

## 技术实现

### 架构

```
wx.RecorderManager（PCM 16000Hz 单声道）
  → onFrameRecorded（每 0.3 秒一帧）
    → RMS 音量计算
    → 简化 DFT 低频分析
    → 双重阈值判断
    → 触发回调
```

### 1. 录音配置

```javascript
recorderManager.start({
  format: 'PCM',
  frameSize: 10,          // 10KB/帧 ≈ 0.3秒，兼顾实时性与性能
  sampleRate: 16000,      // 16kHz 足够捕捉低频
  numberOfChannels: 1     // 单声道，减少计算量
})
```

### 2. RMS 音量计算

```javascript
function calculateRMS(pcmData) {
  let sumSquares = 0
  for (let i = 0; i < pcmData.length; i++) {
    const normalized = pcmData[i] / 32768.0  // Int16 归一化到 -1~1
    sumSquares += normalized * normalized
  }
  return Math.sqrt(sumSquares / pcmData.length)
}
```

### 3. 简化 DFT 低频分析

不需要完整 FFT 库（小程序包体积有限），简化 DFT 足够识别吹气：

```javascript
function calculateLowFreqRatio(pcmData, sampleRate) {
  const fftSize = 1024
  if (pcmData.length < fftSize) return 0

  // 归一化
  const samples = new Float32Array(fftSize)
  for (let i = 0; i < fftSize; i++) {
    samples[i] = pcmData[i] / 32768.0
  }

  // 简化 DFT：只计算频谱幅度
  const spectrum = new Array(fftSize / 2).fill(0)
  for (let k = 0; k < fftSize / 2; k++) {
    let real = 0, imag = 0
    for (let n = 0; n < fftSize; n++) {
      const angle = (-2 * Math.PI * k * n) / fftSize
      real += samples[n] * Math.cos(angle)
      imag += samples[n] * Math.sin(angle)
    }
    spectrum[k] = Math.sqrt(real * real + imag * imag)
  }

  // 计算低频（0-500Hz）占比
  const freqResolution = sampleRate / fftSize  // ≈ 15.6 Hz/bin
  const lowFreqBins = Math.floor(500 / freqResolution)  // ≈ 32 bins

  let lowEnergy = 0, totalEnergy = 0
  for (let i = 0; i < spectrum.length; i++) {
    totalEnergy += spectrum[i]
    if (i < lowFreqBins) lowEnergy += spectrum[i]
  }

  return totalEnergy > 0 ? (lowEnergy / totalEnergy * 100) : 0
}
```

### 4. 单例模式

`RecorderManager` 全局唯一，多个组件共用时必须用单例避免冲突：

```javascript
class AudioDetector {
  constructor() {
    if (AudioDetector.instance) return AudioDetector.instance
    this.isRunning = false
    this.config = { volumeThreshold: 0.1, lowFreqThreshold: 30 }
    this.callbacks = {}
    AudioDetector.instance = this
  }

  configure(config) { this.config = { ...this.config, ...config } }
  setCallbacks(cb) { this.callbacks = { ...this.callbacks, ...cb } }

  start() { /* ... */ }
  stop() { /* ... */ }
}

export default new AudioDetector()  // 导出实例，非类
```

## 鸿蒙兼容：三个坑

### 坑 1：系统降噪吃掉吹气声

华为/荣耀设备的系统级降噪算法会把吹气声视为噪音直接过滤。

**解法**：检测到华为设备时，降低检测阈值。

```javascript
const systemInfo = wx.getSystemInfoSync()
const isHuawei = /huawei|honor|harmonyos/i.test(systemInfo.brand || systemInfo.system || '')

audioDetector.configure({
  volumeThreshold: isHuawei ? 0.05 : 0.1,   // 音量阈值减半
  lowFreqThreshold: isHuawei ? 20 : 30,      // 低频阈值也降低
  frameSize: isHuawei ? 5 : 10,              // 更小的帧（见坑2）
})
```

### 坑 2：onFrameRecorded 不回调

部分鸿蒙设备上 `onFrameRecorded` 完全不触发（怀疑是系统录音服务对分帧的支持不完整）。

**解法**：设置超时检测，超时后降级为摇一摇交互。

```javascript
// 启动后 3 秒内没收到任何帧回调 → 判定为不可用
this._frameTimeout = setTimeout(() => {
  if (this._frameCount === 0) {
    this.callbacks.onError?.({ errorCode: 'frame_timeout' })
    // 提示用户改用摇一摇
  }
}, 3000)
```

### 坑 3：帧大小影响回调触发率

华为设备上 `frameSize: 10`（10KB）可能导致帧回调间隔过长甚至不触发，改为 5KB 后恢复正常。

---

## 降级方案：摇一摇

当吹气检测不可用时（权限拒绝 / 鸿蒙帧超时），降级为加速度计摇一摇：

```javascript
wx.startAccelerometer({ interval: 'game' })
wx.onAccelerometerChange((res) => {
  const { x, y, z } = res
  const magnitude = Math.sqrt(x * x + y * y + z * z)

  if (magnitude > shakeThreshold && Date.now() - lastShakeTime > shakeInterval) {
    lastShakeTime = Date.now()
    this.onBlow()  // 复用同一个触发函数
  }
})
```

## 错误处理

```javascript
function handleError(error) {
  const errMsg = String(error?.errMsg || error?.message || error)

  // 权限错误 → 引导授权，不重试
  if (/auth deny|authorize|permission/i.test(errMsg)) {
    wx.showModal({
      title: '需要麦克风权限',
      content: '请在设置中开启麦克风权限',
      confirmText: '去设置',
      success: (res) => { if (res.confirm) wx.openSetting() }
    })
    return
  }

  // 其他错误 → 自动重试（指数退避，最多 3 次）
  if (retryCount < 3) {
    retryCount++
    setTimeout(() => this.start(), 1000 * retryCount)
  }
}
```

## 可复用结论

1. **吹气 ≠ 音量大**。单纯的音量阈值误触率极高，必须加频率特征判断（低频占比）。
2. **简化 DFT 够用**。不需要引入 FFT 库，1024 点 DFT 就能区分吹气和说话。
3. **鸿蒙降噪极其激进**。华为/荣耀设备必须单独降低阈值，否则用户吹到缺氧也触发不了。
4. **必须有降级方案**。麦克风权限可能被拒绝、帧回调可能不触发，摇一摇是可靠的 Plan B。
5. **单例模式管理录音器**。`RecorderManager` 全局唯一，多组件共用时单例避免冲突。
6. **frameSize 影响鸿蒙兼容**。华为设备上 10KB 帧可能不回调，改 5KB 恢复。
